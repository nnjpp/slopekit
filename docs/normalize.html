Normalize

Reduces inconsistencies in the default styling of HTML elements across all browsers.

This component is based on the famous Normalize.css, a project by Nicolas Gallagher. It takes a different approach to the traditional CSS reset and avoids redundant code by only targeting the styles that need normalizing. The Normalize component is always loaded first, so you don't have to worry about browser inconsistencies in your projects.

Modifications

For UIkit, we made some modifications and improvements in the Normalize.css.

  • Moved <mark> and <h1> defaults to Base component
  • Changed <fieldset> defaults to 0
  • Added a cursor for radio and checkbox elements
  • Set form controls box sizing to border-box
  • Modified :disabled selector
  • Better font-family for <code>, <kbd>, <pre> and <samp>
  • Removed placeholder transparency in Firefox
  • Removed hgroup because it is deprecated in HTML5
  • Kept vertical-align set to top
